## Configure DynamoDB for ROMP

Create a DynamoDB table named **ROMP-AlertHistory** to store alert history generated by the monitoring modules (Inventory, Expiry, Promotion, and Sales) using the following configuration.

![create stack](/images/5-Workshop/5.2-Prerequisite/DDB1.jpg)

* In the AWS Console search bar, type **DynamoDB** and open the **DynamoDB** service.

* From the left navigation menu, select **Tables**.

* Click **Create table**.

---

![create stack](/images/5-Workshop/5.2-Prerequisite/DDB2.jpg)

Configure the table:

* **Table name**

```
ROMP-AlertHistory
```

* **Partition key**

```
AlertID
```

* Data type:

```
String
```

* Leave the **Sort key** empty.

---

![create stack](/images/5-Workshop/5.2-Prerequisite/DDB3.jpg)

Under **Table settings**:

* Select **Customize settings**.

* Under **Table class**, keep the default option:

```
DynamoDB Standard
```

---

![create stack](/images/5-Workshop/5.2-Prerequisite/DDB4.jpg)

Under **Capacity mode**, select:

```
On-demand
```

Do **not** choose **Provisioned** capacity.

---

![create stack](/images/5-Workshop/5.2-Prerequisite/DDB5.jpg)

Optionally, add the following tag for easier resource and cost management:

```
Project = ROMP
```

Review all configuration settings, then click **Create table**.

---

![create stack](/images/5-Workshop/5.2-Prerequisite/DDB6.jpg)

Wait a few seconds until the table status changes from:

```
Creating
```

to

```
Active
```

# Configure Amazon SES for ROMP

Verify an email address in Amazon SES so that AWS Lambda can send alert emails to your mailbox.

---

![create stack](/images/5-Workshop/5.2-Prerequisite/SES1.jpg)

From the left navigation menu, select:

```
Identities
```

under the **Configuration** section.

Click **Create identity**.

---

![create stack](/images/5-Workshop/5.2-Prerequisite/SES2.jpg)

Configure the identity:

* **Identity type**

```
Email address
```

* **Email address**

Enter your own valid email address.

---

![create stack](/images/5-Workshop/5.2-Prerequisite/SES3.jpg)

Scroll down to the **Tags** section.

This step is optional.

You may skip it or add the following tag:

```
Project = ROMP
```

Click **Create identity**.

---

![create stack](/images/5-Workshop/5.2-Prerequisite/SES4.jpg)

AWS will immediately send a verification email to the address you provided.

The email subject will be similar to:

```
Amazon Web Services – Email Address Verification Request
```

Open the email and click the verification link inside.

---

![create stack](/images/5-Workshop/5.2-Prerequisite/SES5.jpg)

Your browser will open a confirmation page displaying a message similar to:

```
Congratulations! You have successfully verified...
```

This confirms that the email address has been successfully verified.

---

![create stack](/images/5-Workshop/5.2-Prerequisite/SES6.jpg)

Return to the **Identities** page in the Amazon SES Console and refresh the page.

The identity status should change from:

```
Pending verification
```

to

```
Verified
```
