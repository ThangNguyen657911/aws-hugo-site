Here is the list and introduction of the blogs you have published on [AWS Study Group](https://www.facebook.com/groups/awsstudygroupfcj):

###  [Blog 1 - Optimizing Big Data Cost and Reliability: Deploying Remote Shuffle Service on Amazon EMR with Apache Celeborn](3.1-Blog1/)
This blog shares a solution to optimize costs and enhance reliability for large-scale data processing systems (Apache Spark) on AWS. By decoupling storage and compute resources through the deployment of Apache Celeborn as a Remote Shuffle Service (RSS), the system can confidently run 100% on EC2 Spot Instances without worrying about shuffle data loss when nodes are reclaimed.

###  [Blog 2 - Automating Patch Testing on Amazon Redshift: The Ultimate Solution to Protect Production Workloads](3.2-Blog2/)
This blog introduces a solution for building an automated patch testing pipeline on Amazon Redshift. This solution helps businesses simulate and evaluate the impact of database management system updates directly on a replica of production workloads, ensuring absolute integrity and peak performance before actual deployment.

###  [Blog 3 - Building a Large-Scale Real-Time Anomaly Detection System with Razorpay and Amazon MSK](3.3-Blog3/)
This blog shares the architecture of Razorpay's ADA (Anomaly Detection and Alerting) system built on Amazon MSK (Apache Kafka) and Apache Flink. The system processes over 5 billion events daily, detecting and preventing financial fraud in real-time in under 30 seconds, while cutting monitoring infrastructure costs by up to 80%.
